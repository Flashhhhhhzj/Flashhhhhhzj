## About Me

- I enjoy building practical things: web apps, developer workflows, and tools that reduce repetitive work.
- My recent work sits around document automation, Markdown pipelines, and LLM-assisted knowledge processing.
- I care about readable docs, clean structure, steady iteration, and products that solve real problems.
- Outside pure coding, I also like shaping the whole experience: project framing, content clarity, and developer ergonomics.

## Current Focus

- Full-stack projects that start from coursework and grow into polished, real-world builds.
- Automation around Feishu or Lark docs, Markdown conversion, and structured content reuse.
- Shipping more in public and turning experiments into repositories people can actually learn from.

## Selected Projects

| Project | What it does | Status |
| --- | --- | --- |
| [feishu-docx-to-markdown](https://github.com/Flashhhhhhzj/feishu-docx-to-markdown) | Converts Feishu or XFChat cloud documents into clean Markdown while preserving headings, numbering, tables, code blocks, images, and links. | Public |
| `care-course-import-poc` | Parses Feishu wiki content and uses LLM extraction to prefill lesson or course forms for knowledge import workflows. | Private / in migration |
| `skill-runoob2md` | Pulls Runoob tutorial content into a Markdown knowledge repo with structure preserved for reuse. | Private |
| [TreasureGo](https://github.com/St0ff3l/TreasureGo-USM-CAT304W-CSE341W-Project) | A second-hand marketplace platform with listing, search, filtering, and purchase flows. | Public |
| [SleepGo](https://github.com/St0ff3l/SleepGo-USM-CAT201W-Assignment2) | A responsive company homepage project built with React.js and Node.js. | Public |
